## 二、实习阶段安排与学习路线

### 第一阶段：ROS1 基础与开发环境搭建（时间安排：第 1 周）

**1. 学习目标：**

掌握 ROS1 基础开发环境，理解 ROS1 通信机制与常用工具，能够独立创建工作空间、功能包并完成基础节点通信；同时掌握 Git 与 GitHub 项目管理流程，能够将 ROS1 项目纳入版本控制并提交至 GitHub 远程仓库。

**2. 理论内容：**

- ROS1 计算图结构。
- Node、Topic、Service、Action 的通信机制。
- TF 坐标变换原理。
- ROS 工作空间与功能包结构。
- URDF/SDF 机器人描述模型基础。
- Git 基本概念，包括工作区、暂存区、本地仓库和远程仓库。
- GitHub 远程代码托管与协作管理流程。
- Git 基本提交流程，包括 add、commit、push、pull。
- Git 分支管理方法，包括 main/master 分支、dev 分支和 feature 分支。
- README.md、.gitignore、Issue、Pull Request 在项目管理中的作用。

**3. 技术内容：**

- Python 基础，掌握 conda 虚拟环境管理。
- C++ 编译基础，掌握 CMake 项目构建与编译。
- catkin 工作空间创建、编译与管理。
- ROS 功能包创建与节点编写。
- roslaunch、rosparam、rosbag 等工具使用。
- RViz 可视化与调试。
- 安装并配置 Git。
- 在 GitHub 创建远程仓库。
- 在本地初始化 Git 仓库。
- 将 ROS1 项目绑定到 GitHub 远程仓库。
- 编写适用于 ROS1 项目的 .gitignore 文件，避免提交 build、devel、log 等中间文件。
- 编写 README.md，说明项目简介、环境依赖、编译方法、运行方式和演示结果。
- 使用 Git 完成代码提交、分支管理和远程推送。

**4. GitHub 基本提交流程：**

1. 在 GitHub 创建远程仓库。
2. 在本地项目目录初始化仓库：
   ```
   git init
   ```
3. 添加远程仓库地址：
   ```
   git remote add origin https://github.com/用户名/仓库名.git
   ```
4. 查看文件修改状态：
   ```
   git status
   ```
5. 添加需要提交的文件：
   ```
   git add .
   ```
6. 提交到本地仓库：
   ```
   git commit -m "add initial ros project"
   ```
7. 设置主分支名称：
   ```
   git branch -M main
   ```
8. 推送到 GitHub：
   ```
   git push -u origin main
   ```

后续更新代码时，重复以下流程：
```
git status
git add .
git commit -m "update moveit config"
git push
```

**5. 阶段产出：**

- ROS1 环境配置说明文档。
- Python 与 C++ 基础通信 Demo。
- Topic、Service、Action 示例程序。
- TF 坐标变换可视化 Demo。
- ROS1 基础学习笔记。
- GitHub 远程仓库链接。
- 适用于 ROS1 项目的 .gitignore 文件。
- README.md 项目说明文档。
- 不少于 5 次有效 commit 记录。
- 至少 1 个 feature 分支开发记录。
